ECG Arrhythmia Detection

Patient-independent abnormal heartbeat detection using ECG morphology and RR interval dynamics

Project Overview

This project develops a deep learning system for ECG arrhythmia detection using the MIT-BIH Arrhythmia Database. The model combines heartbeat morphology from ECG waveforms with RR interval timing features to classify heartbeats as normal or abnormal.

Unlike a simple random beat split, this project uses a patient-independent DS1/DS2-style evaluation protocol, where the model is tested on ECG records from unseen patients.

Motivation

Arrhythmia detection is not only a waveform classification problem. Beat timing and rhythm dynamics are also important. This project investigates how ECG morphology and RR intervals complement each other for abnormal heartbeat detection.

Methods

  • Dataset: MIT-BIH Arrhythmia Database
  • Input 1: ECG beat window centered around annotated beat locations
  • Input 2: RR interval features: previous RR and next RR intervals
  • Model: Multi-input neural network with a 1D CNN ECG branch and dense RR branch
  • Evaluation: Patient-independent DS1/DS2 split

Model Architecture

The ECG branch learns beat morphology using one-dimensional convolutional layers. The RR branch learns timing information from neighboring beat intervals. The extracted ECG and RR features are concatenated and passed to a final classifier. 

ECG beat window  ──► 1D CNN ──► ECG features
RR intervals     ──► Dense NN ─► RR features

ECG features + RR features ──► Classifier ──► Normal / Abnormal

Key Results

Model Test Accuracy Test ROC-AUC
ECG morphology only 84.2% 0.817
RR intervals only 93.3% 0.832
ECG + RR intervals 94.8% 0.957

The fused ECG + RR model substantially outperformed both ECG-only and RR-only baselines, showing that heartbeat morphology and rhythm timing provide complementary information.

Clinical and Technical Relevance

This work demonstrates a near-real-time abnormal beat detection pipeline. In a real streaming system, incoming ECG data would be processed through R-peak detection, beat extraction, RR interval calculation, and model inference to flag normal or abnormal beats.
